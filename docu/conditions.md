## 卖出条件 (`judge_sell_id`)

### `sell_id = 1`：标准卖出策略 1

此 ID 包含一套基于多时间周期和技术指标的综合卖出策略，按以下优先级顺序执行：

1.  **清仓信号**：
    - **条件**：周线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且 120 分钟 K 线图上出现 MACD 死叉（3 天内）。
    - **操作**：建议清仓。

1.  **清仓信号**：
    - **条件**：月线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且 120 分钟 K 线图上出现 MACD 死叉（3 天内）。
    - **操作**：建议清仓。

2.  **减仓信号**：
    - **条件**：日线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且 120 分钟 K 线图上出现 MACD 死叉（3 天内）。
    - **操作**：建议卖出 1/3 仓位。

3.  **阶梯式减仓信号**：
    - **触发条件**：日线 SAR 指标被跌破（3 天内）。
    - **操作**：根据周线 RSI 指标的强度执行不同比例的卖出：
        - **周线 RSI > 90**：建议卖出全部剩余仓位。
        - **周线 RSI > 85**：建议卖出剩余仓位的 1/2。
        - **周线 RSI > 80**：建议卖出 1/3 仓位。

### `sell_id = 2`：标准卖出策略 2

此 ID 包含一套基于多时间周期和技术指标的综合卖出策略，按以下优先级顺序执行：

## 买入条件 (`judge_buy_id` / `judge_t_id`)

系统支持不同的买入策略，通过 `judge_buy_id`（新开仓）或 `judge_t_id`（做 T 买回）进行选择。两者复用相同的逻辑 ID，但提示信息会有所区别：
- **`judge_buy_id`**：提示 **"新资金买入信号"**。
- **`judge_t_id`**：提示 **"做 T 买回信号"**（用于买回之前卖出的资金）。

### `buy_id` = 1：保守型买入
- **条件**：日线 RSI < 20 **并且** 周线 RSI < 25。

### `buy_id` = 2：标准型买入 / 做 T 买回
- **说明**：此 ID 常用于判断卖出后的资金买回时机。
- **条件**：日线 RSI < 25 **并且** 周线 RSI < 30。

### `buy_id` = 3：指数保护型买入
- **说明**：此 ID 用于监控市场大盘是否处于超卖状态。
- **条件**：**创业板指数 (399006)** 的日线 RSI < 25。

### `buy_id` = 4：极度保守型买入
- **条件**：日线 RSI < 20 **并且** 周线 RSI < 20。

### `buy_id` = 5：指数保护型买入
- **说明**：此 ID 用于监控市场大盘是否处于超卖状态。
- **条件**：**创业板指数 (399006)** 的日线 RSI < 20。

## RSI 相关指标判断逻辑

### RSI flag 状态跟踪

**初始化**：`rsi_flag = 0`

**状态更新规则**：
1. 周线 RSI > 80 → `rsi_flag = 1`（一级警戒）
2. 周线 RSI > 85 → `rsi_flag = 2`（二级警戒）
3. 周线 RSI > 90 → `rsi_flag = 3`（三级警戒/清仓信号）
4. 当 5 日均线跌破 10 日均线时 → `rsi_flag = 0`（重置）

**说明**：
- RSI flag 反映的是从上次均线跌破后到当前的周线 RSI 峰值等级
- 均线跌破后，flag 重置为 0，并重新根据新的 RSI 峰值更新

### RSI 位置状态（位置 1-5 标记）

**位置标记规则**：
- **位置 0**：初始状态，等待 RSI>90
- **位置 1**：RSI>90 时的股价高点（局部高点）
- **位置 2**：从位置 1 下跌后的低点（局部低点）
- **位置 3**：不是局部高点（因为从位置 1 只下跌了 2%，不满足显著回调）
- **位置 4**：RSI>90 后下跌 2% 的位置（确认局部低点）
- **位置 5**：从位置 4 上涨 10% 的位置（反弹高点，用于顶背离判断）

**状态转换示例**：
```
初始 (0) → RSI>90 → 位置 1 → RSI 下跌 2% → 位置 4 → RSI 上涨 10% → 位置 5
```

**局部高点判断示例**：
```
上涨过程中，当股价下跌 5% 之后，可以认为此时之前的高点是一个局部高点。

示例流程：
1. 股价先上涨了 10% → 这个位置记为 1（局部高点候选）
2. 然后下跌了 5% → 这个位置记为 2（确认位置 1 是局部高点）
3. 然后又上涨了 20% → 这个位置记为 3（新的局部高点候选）
4. 再下跌了 2% → 这个位置记为 4（位置 3 不是局部高点，因为只下跌了 2%）
5. 再上涨了 10% → 这个位置记为 5

结论：位置 1 是局部高点，位置 2 是局部低点，位置 3 不是局部高点
```

**判断条件**：
- 前一天的 5 日均线大于 10 日均线
- 当天的 5 日均线小于 10 日均线

## 日线顶背离判断逻辑

**顶背离形成条件**：
1. 当前最高点股价比之前局部最高点股价高
2. MACD 红柱子比之前局部最高点 MACD 柱子短（或绿柱子变长）
3. 两个高点之间间隔至少 5 根 K 线（确保是两个独立波段）
4. 此时 `diverse_day_flag = 1`

**顶背离失效（假背离）**：
- 条件：后续股价继续新高，且 MACD 柱子比之前局部最高点的 MACD 高
- 操作：`diverse_day_flag = 0`

**顶背离生效**：
- 条件：5 日均线跌破 10 日均线
- 操作：`diverse_day_flag = 0`，同时卖出规定比例的金额（通常 1/3）

---

## 周线顶背离判断逻辑

**顶背离形成条件**：
1. 当前最高点股价比之前局部最高点股价高
2. MACD 红柱子比之前局部最高点 MACD 柱子短
3. 两个高点之间间隔至少 5 周（确保是两个独立波段）
4. 此时 `diverse_week_flag = 1`

**顶背离失效（假背离）**：
- 条件：后续股价继续新高，且 MACD 柱子比之前局部最高点的 MACD 高
- 操作：`diverse_week_flag = 0`

**顶背离生效**：
- 条件：5 周均线跌破 10 周均线
- 操作：`diverse_week_flag = 0`，同时卖出规定比例的金额（通常 50%）

## 顶背离判断依据

对于日线顶背离和周线顶背离的判断，满足以下条件符合顶背离：

1. **股价创新高**：不是当前股价，而是那一天的最高价（High）创新高
2. **MACD 柱子缩短**：红柱子缩短或绿柱子变长（hist 下降）
3. **间隔要求**：若两次高点间隔太短（<5 根 K 线），可能是震荡，不算顶背离
4. **确认条件**：5 日均线跌破 10 日均线，才能确认顶背离有效

**局部高点判断**：
- 上一个股价高点是一个局部高点
- 只有当股价下跌 5% 左右，才能判断之前的高点是局部高点
- 后续再拿最新的股价高点去和上一次局部高点作比较：
  - （1）比较股价：是否比之前的局部高点更高
  - （2）比较 MACD：判断是否背离

## 其他规则

1. **卖出后 5 个交易日内不再卖出**：避免频繁交易

2. **judge_t_id vs judge_buy_id**：
   - `judge_t_id`：买回之前卖出的资金，新资金不买入
   - `judge_buy_id`：新资金入场，如果之前有卖出资金，也同样买回

3. **回测与实时监控的区别**：
   - **回测** (`gm_backtest.py`)：
     - 回测当天触发告警直接买卖交易即可
     - 不需要检查过去三天触发告警买卖点三次
     - 每天遍历所有交易日，逐日进行买卖判断

   - **实时监控** (`gm_live.py`)：
     - 过去三天触发买卖点都要告警出来
     - 例如：周一触发，但周一、周二没有运行脚本，周三运行脚本时依然要能告警出来
     - 检查过去 7 天的数据，确保不会遗漏任何信号
     - 状态（rsi_flag、rsi_state）只在检查完所有历史数据后，用最新数据更新一次

4. **定时任务设置**：
   - 每天 14:50 自动运行脚本（接近收盘时）
   - 如果 14:50 时电脑休眠，晚上 19:00 电脑启动后，脚本会自动运行一次
   - 初始化时会立即执行一次检查，确认程序正常运行
   - **实时监控**：过去三天触发买卖点都要告警出来

1、当前后续股价继续新高，如果这时候MACD柱子比之前局部最高点的MACD高，此时认为之前顶背离失效，可以认为是假背离，diverse_week_flag = 0
2、当前后续股价5日均线跌破10日均线，此时认为顶背离生效，diverse_week_flag置为0，同时卖出规定比例的金额
