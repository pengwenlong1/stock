# 买卖条件文档

## 卖出条件 (`judge_sell_id`)

### `sell_id = 1`：标准卖出策略 1

此 ID 包含一套基于多时间周期和技术指标的综合卖出策略，按以下优先级顺序执行：

1.  **清仓信号 - 周线顶背离**：
    - **条件**：周线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且5日均线跌破10日均线。
    - **操作**：建议清仓。

2.  **清仓信号 - 月线顶背离**：
    - **条件**：月线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且5日均线跌破10日均线。
    - **操作**：建议清仓。

3.  **清仓信号 - RSI 极度超买**：
    - **条件**：周线 RSI > 90（rsi_flag=3），并且5日均线跌破10日均线。
    - **操作**：建议清仓。

4.  **减仓信号 - 日线顶背离**：
    - **条件**：日线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且5日均线跌破10日均线。
    - **操作**：建议卖出 1/3 仓位。

5.  **阶梯式减仓信号**：
    - **触发条件**：5日均线跌破10日均线。
    - **操作**：根据周线 RSI 指标的强度执行不同比例的卖出：
        - **周线 RSI > 85**：建议卖出剩余仓位的 1/2。
        - **周线 RSI > 80**：建议卖出 1/3 仓位。

**重要说明**：
- 当多个卖出信号同时触发时，执行最大卖出比例
- RSI flag=3（周线RSI>90）触发清仓时，优先级高于日线顶背离
- 周线顶背离检测使用 grace_bars=5，允许信号在最近5周内持续有效

### `sell_id = 2`：标准卖出策略 2

此 ID 包含一套基于多时间周期和技术指标的综合卖出策略，按以下优先级顺序执行：

1.  **清仓信号 - 周线顶背离**：
    - **条件**：周线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且5日均线跌破10日均线。
    - **操作**：建议清仓。

2.  **清仓信号 - 月线顶背离**：
    - **条件**：月线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且5日均线跌破10日均线。
    - **操作**：建议清仓。

3.  **清仓信号 - RSI 极度超买**：
    - **条件**：周线 RSI > 90（rsi_flag=3），并且5日均线跌破10日均线。
    - **操作**：建议清仓。

4.  **减仓信号 - 日线顶背离**：
    - **条件**：日线出现 MACD 顶背离（参考下方"顶背离判断依据"），并且5日均线跌破10日均线。
    - **操作**：建议卖出 1/3 仓位。

5.  **阶梯式减仓信号**：
    - **触发条件**：5日均线跌破10日均线。
    - **操作**：根据周线 RSI 指标的强度执行不同比例的卖出：
        - **周线 RSI > 85**：建议卖出剩余仓位的 1/2。
        - **周线 RSI > 80**：建议卖出 1/3 仓位。

## 买入条件 (`judge_buy_id` / `judge_t_id`)

系统支持不同的买入策略，通过 `judge_buy_id`（新开仓）或 `judge_t_id`（做 T 买回）进行选择。两者复用相同的逻辑 ID，但提示信息会有所区别：
- **`judge_buy_id`**：提示 **"新资金买入信号"**。
- **`judge_t_id`**：提示 **"做 T 买回信号"**（用于买回之前卖出的资金）。

### `buy_id` = 1：保守型买入
- **条件**：日线 RSI < 20 **并且** 周线 RSI < 25。

### `buy_id` = 2：标准型买入 / 做 T 买回
- **说明**：此 ID 常用于判断卖出后的资金买回时机。
- **条件**：日线 RSI < 25 **并且** 周线 RSI < 30。

### `buy_id` = 3：指数保护型买入
- **说明**：此 ID 用于监控市场大盘是否处于超卖状态。
- **条件**：**创业板指数 (399006)** 的日线 RSI < 25。

### `buy_id` = 4：极度保守型买入
- **条件**：日线 RSI < 20 **并且** 周线 RSI < 20。

### `buy_id` = 5：指数保护型买入
- **说明**：此 ID 用于监控市场大盘是否处于超卖状态。
- **条件**：**创业板指数 (399006)** 的日线 RSI < 20。

## RSI 相关指标判断逻辑

### RSI flag 状态跟踪

**初始化**：`rsi_flag = 0`

**状态更新规则**：
1. 周线 RSI > 80 → `rsi_flag = 1`（一级警戒）
2. 周线 RSI > 85 → `rsi_flag = 2`（二级警戒）
3. 周线 RSI > 90 → `rsi_flag = 3`（三级警戒/清仓信号）
4. 当 5 日均线跌破 10 日均线时 → `rsi_flag = 0`（重置）

**说明**：
- RSI flag 反映的是从上次均线跌破后到当前的周线 RSI 峰值等级
- 均线跌破后，flag 重置为 0，并重新根据新的 RSI 峰值更新

### RSI 位置状态（位置 1-5 标记）

**位置标记规则**：
- **位置 0**：初始状态，等待 RSI>90
- **位置 1**：RSI>90 时的股价高点（局部高点）
- **位置 2**：从位置 1 下跌后的低点（局部低点）
- **位置 3**：不是局部高点（因为从位置 1 只下跌了 2%，不满足显著回调）
- **位置 4**：RSI>90 后下跌 2% 的位置（确认局部低点）
- **位置 5**：从位置 4 上涨 10% 的位置（反弹高点，用于顶背离判断）

**状态转换示例**：
```
初始 (0) → RSI>90 → 位置 1 → RSI 下跌 2% → 位置 4 → RSI 上涨 10% → 位置 5
```

### 局部高点判断（无未来函数实现方式）

**核心原则**：判断只能在历史数据上进行，不能使用当前K线之后的数据。

**实现逻辑**：
1. 遍历每一根K线（索引 i）
2. 记录最近一个候选高点 `candidate_high_price` 和其位置 `candidate_high_index`
3. 当价格从候选高点下跌 ≥ 5% 时（在下跌发生的那根K线）：
   - 确认 `candidate_high` 是一个有效的局部高点
   - 将 `candidate_high` 存入局部高点列表
   - 重置候选高点为当前价格
4. 如果价格在下跌5%之前创了新高：
   - 更新 `candidate_high_price` 和 `candidate_high_index` 为新高

**注意**：局部高点的"确认时刻"是下跌5%发生的那根K线，而不是最高点出现的那根K线。

**判断条件**：
- 前一天的 5 日均线大于 10 日均线
- 当天的 5 日均线小于 10 日均线
- 满足以上两个条件则判断为 5 日均线跌破 10 日均线

# 顶背离判断逻辑（部分方法是日线/周线通用）

## 一、日线局部高点检测（无未来函数）

- 维护一个“候选高点”（价格、位置、MACD柱值）。
- 当价格创新高时，更新候选高点。
- 当价格从候选高点下跌达到 **5%** 时：
  - 确认该候选高点是**有效的局部高点**。
  - 将高点存入局部高点列表（记录日期、价格、MACD柱值）。
  - 将当前价格设为新的候选高点。
- **关键点**：局部高点的“确认时刻”是下跌5%发生的那根K线，而非最高点出现的那根K线。
- 
## 一、周线局部高点检测（无未来函数）

- 维护一个“候选高点”（价格、位置、MACD柱值）。
- 当价格创新高时，更新候选高点。
- 当价格从候选高点下跌达到 **8%** 且 5日均线跌破10日均线时：
  - 确认该候选高点是**有效的周线级别局部高点**。
  - 将高点存入局部高点列表（记录日期、价格、MACD柱值）。
  - 将当前价格设为新的周线级别候选高点。
- **关键点**：周线局部高点的“确认时刻”是下跌8%且5日均线跌破10均线 发生的那根K线，而非最高点出现的那根K线。

## 二、顶背离形成

**判断时机**：每次确认一个新的局部高点 **B** 时。

**判断方法**：
1. 从已确认的局部高点列表中，**从最近向更早遍历**，寻找第一个满足以下条件的高点 **A**：
   - A 的日期比 B 早；
   - B 与 A 的K线间隔：日线 ≥ 5根，周线 ≥ 5周；
   - B 的最高价 > A 的最高价；
   - B 的MACD柱值 < A 的MACD柱值。
2. 如果找到这样的 A，则形成一次**顶背离**。

**记录与标志**：
- 记录背离形成时间、高点A和B的价格与MACD值。
- 设置背离标志 = 1。
- **连续背离处理**：如果已有背离标志为1，又形成新的背离（C与某个A'），且新背离的强度更高（C价格更高、MACD更低），则更新背离记录为最新一对，覆盖旧背离。

## 三、背离失效条件

满足以下**任一**条件，背离标志清零：

1. **强信号失效**：出现新的局部高点 C，且 C 的价格 > 当前背离高点 B 的价格，并且 C 的MACD柱值 > B 的MACD柱值（意味着趋势未衰竭，假背离）。
2. **时间衰减失效**：背离形成后，超过 **10个交易日（日线）** 或 **10周（周线）** 仍未触发均线跌破确认。
3. **反向趋势失效**：价格从背离高点 B 下跌超过 **10%** 且未能反弹（视为趋势逆转）。

## 四、背离生效确认

**日线顶背离生效条件**（同时满足）：
- 日线背离标志 = 1
- 5日均线跌破10日均线

**生效后操作**：
- 卖出 1/3 仓位
- 日线背离标志清零

**周线顶背离生效条件**（同时满足）：
- 周线背离标志 = 1
- 5日均线跌破10日均线（注意：不是周均线）

**生效后操作**：
- 清仓（卖出全部仓位）
- 周线背离标志清零

## 顶背离计算参数配置

| 参数 | 日线值 | 周线值 | 说明 |
|------|--------|--------|------|
| 局部高点确认跌幅 | 5% | 5% | 从候选高点下跌多少确认有效 |
| 最小间隔K线数 | 5根 | 5周 | 两个高点之间的最小距离 |
| 背离失效时间 | 10天 | 10周 | 超过此时长未确认则失效 |
| 价格比较基准 | 最高价 | 最高价 | 比较高点的 High |
| MACD比较基准 | 柱值(Hist) | 柱值(Hist) | 取高点所在K线的柱值 |

---

## 重要说明

1. **回溯比较**：每次新确认高点时，需与**历史所有已确认高点**比较（而不仅仅是前一个），找到最近一个满足背离条件的配对，避免漏报。
2. **均线确认**：日线和周线顶背离的最终确认均使用**日线级别的5/10日均线跌破**，保持一致。
3. **状态独立**：日线背离和周线背离的状态标志（diverse_day_flag / diverse_week_flag）独立维护，互不干扰。

## 其他规则

1. **卖出后 5 个交易日内不再卖出**：避免频繁交易

2. **judge_t_id vs judge_buy_id**：
   - `judge_t_id`：买回之前卖出的资金，新资金不买入
   - `judge_buy_id`：新资金入场，如果之前有卖出资金，也同样买回

3. **回测与实时监控的区别**：
   - **回测** (`gm_backtest.py`)：
     - 回测当天触发告警直接买卖交易即可
     - 不需要检查过去三天触发告警买卖点三次
     - 每天遍历所有交易日，逐日进行买卖判断

   - **实时监控** (`gm_live.py`)：
     - 过去三天触发买卖点都要告警出来
     - 例如：周一触发，但周一、周二没有运行脚本，周三运行脚本时依然要能告警出来
     - 检查过去 7 天的数据，确保不会遗漏任何信号
     - 状态（rsi_flag、rsi_state）只在检查完所有历史数据后，用最新数据更新一次

4. **定时任务设置**：
   - 每天 14:50 自动运行脚本（接近收盘时）
   - 如果 14:50 时电脑休眠，晚上 19:00 电脑启动后，脚本会自动运行一次
   - 初始化时会立即执行一次检查，确认程序正常运行
   - **实时监控**：过去三天触发买卖点都要告警出来

## 日志打印要求

### 卖出日志格式

**基本要求**：
1. **卖出时间点**：显示卖出发生的日期
2. **卖出依据策略**：显示依据哪一个策略（策略 1-阶梯、策略 1-顶背离均线、策略 1-周线顶背等）
3. **RSI 详细信息**：如果依据 RSI 策略，需要显示：
   - 触发哪一天的 RSI 峰值
   - RSI 峰值的值
4. **顶背离详细信息**：如果依据顶背离策略，需要显示：
   - 顶背离形成的日期
   - 顶背离时的前高价格

回测脚本描绘一下K线图，绘画出买点，卖点，买卖价格，买卖原因，买卖如果是因为RSI需要显示RSI的值。

### K线图绘制功能

回测完成后会自动生成K线图，保存在 `main/logs/` 目录下，文件名格式为 `{股票名}_backtest_kline.png`。

**K线图包含以下内容**：
1. **K线图**：红色表示上涨，蓝色表示下跌
2. **买卖点标记**：
   - 绿色箭头向上：买入点
   - 红色箭头向下：卖出点
3. **买卖信息标注**：
   - 买卖价格
   - 日RSI值和周RSI值（如果有）
   - 买卖原因（简化版）
4. **RSI指标图**：
   - 日RSI曲线
   - 超买线(80)和超卖线(20)
   - 超买超卖区域阴影标记

**买卖原因简化显示对照**：

| 完整原因关键字 | 简化显示 |
|--------------|---------|
| 阶梯 + flag=3 | RSI>90清仓 |
| 阶梯 + flag=2 | RSI>85减半 |
| 阶梯 + flag=1 | RSI>80减1/3 |
